Práctica BBDD - Gestión de Veterinaria

Necesidad de negocio
Diseñé esta base de datos para una clínica veterinaria que necesita organizar su información. El objetivo es dejar de usar papel y tener controlados a los clientes, sus mascotas, las citas que tienen y los tratamientos que se les ponen en cada visita.

Pasos seguidos
Para este trabajo seguí este proceso:

Diseño de tablas: Cree 5 tablas: Clientes, Mascotas, Veterinarios, Citas y Tratamientos.


Campos: Puse un mínimo de 4 campos por cada tabla para que no falte información importante (nombres, teléfonos, fechas, etc.).


Relación intermedia: Incluí la tabla Cita_Tratamiento, que funciona como tabla intermedia para conectar las citas con los medicamentos recetados.


Código SQL: Redacté el script para MariaSQL y lo probé en mi XAMPP local para ver que todo funciona y se crean las relaciones correctamente.


Subida a GitHub: Cree el repositorio script_clinica.sql y subí el script junto con este archivo explicativo.
